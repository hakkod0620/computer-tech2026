UDP（User Datagram Protocol）是一種無連接的「發了就忘」傳輸協定，不保證封包送達，因此掉包（丢包）常見於網路擁塞、頻寬不足、防火牆阻擋或收發端處理能力不足。因其不具備重傳機制，應用層需自行設計確認機制。 
UDP 掉包原因與查測方向：
網路因素：
網路擁塞： 頻寬飽和導致交換機丟棄封包。
實體線路： 乙太網路纜線故障或老化。
路由器問題： 路由器負載過高或連接埠故障。
本機與防火牆因素：
防火牆/ACL阻擋： Windows 防火牆或硬體防火牆阻擋了未定義的入站 UDP 流量。
接收緩衝區溢位： 應用程式處理速度趕不上網路收包速度，導致 socket 緩衝區滿而丟包。
